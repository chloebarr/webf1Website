Week One

This week we looked at the VM, I will need to research what a VM is. A virtual machine is a computer file, typically called an image, that behaves like an actual computer. In other words, a computer is created within a computer. It runs in a window, much like any other program, giving the end user the same experience on a virtual machine as they would have on the host operating system itself. (https://azure.microsoft.com/en-gb/overview/what-is-a-virtual-machine/) The virtual machine will be helpful to learn Linux commands on and create web servers for web development.

Week Four

This week we looked at how to write in an academic style. What is academic writing, Academic writing is a particular style used in formal essays and other assessments for your course. It requires formal language, a logical structure and should be supported by evidence. Most sentences in a literature review should contain a citation, and none of the work should be one's opinion but a summary of other peoples research.

Week Five

This week we were introduced to HTML. What is HTML, Hypertext Markup Language, a standardized system for tagging text files to achieve font, colour, graphics, and hyperlink effects on World Wide Web pages. All HTML files have to be started with (!doctype html).

  • (head) defines the information about the document
  • (title) defines the title for a document
  • (body) defines a documents body, normally the main part of the page
  • (h1) to (h6) defines html headings
  • (p) defines a paragraph
  • (img) defines an image
  • (picture) defines a container for multiple image resources
  • (audio) defines sound content
  • (video) defines a video or movie
  • (track) defines a text track for media elements
  • (a) defines a hyperlink
  • (nav) defines navigation links
  • (ul) defines an unordered list
  • (ol) defines an ordered list
  • (li) defines a list item
  • (header) defines a header for a document or section
  • (footer) defines a footer for a document or section
  • (main) defines the main content of a document

Week Nine

This week we looked at design, and using the MVC design pattern. The MVC design pattern (model, view and controller) has three considerations; information design, technical and architectural design and interaction design. Information design model is about whats on your website and why. Interaction design view is how your website will be laid out, what colours will be used, etc. interaction design controller is providing consistent navigation tools across the site, consistency of colours, etc. technical and architectural design is the file system design, web standards and access control.
